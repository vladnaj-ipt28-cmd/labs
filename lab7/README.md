##  Опис проєкту та проблематика
Цей проєкт є фінальною еволюцією програми для обробки відеопотоку. Головна мета лабораторної роботи — навчитися поєднувати класичне програмування на C++ із сучасними методами Machine Learning / Computer Vision, а також вирішити фундаментальну проблему "важких" обчислень у реальному часі.

**Проблема:** Інференс (робота) глибокої нейромережі для пошуку облич займає суттєвий час (від 30 до 100+ мс на кадр залежно від "заліза"). Якщо виконувати детекцію в основному циклі (`while`), програма не встигатиме оновлювати вікно, відео стане "смиканим", а FPS впаде до некомфортних значень, повністю блокуючи чутливість інтерфейсу користувача.

**Рішення:** Застосування асинхронної багатопотоковості. Ми відокремлюємо процес отримання та відображення кадрів (Main Thread) від процесу пошуку облич (Worker Thread). 

## Робота з Deep Neural Networks (DNN)
Для розпізнавання облич у проєкті використовується сучасний детектор на базі архітектури **ResNet-10** (Single Shot MultiBox Detector - SSD). 
Для ініціалізації мережі використовуються два файли:
* `deploy.prototxt` — текстовий файл, що описує архітектуру та шари нейромережі.
* `res10_300x300_ssd_iter_140000.caffemodel` — бінарний файл із заздалегідь натренованими вагами мережі.

Кадр перед подачею в мережу перетворюється у спеціальний формат (Blob) розміром 300x300 пікселів за допомогою функції `cv::dnn::blobFromImage`. Відфільтровуються лише ті обличчя, рівень впевненості (confidence) яких перевищує 50%.

## Архітектура класів та Багатопотоковість
Програма побудована за модульним ООП-принципом і складається з наступних компонентів:

1. **`CameraProvider`** — забезпечує стабільне захоплення сирих кадрів (`cv::Mat`) з веб-камери.
2. **`KeyProcessor`** — обробляє натискання клавіш та відповідає за зміну станів програми (enum-режими).
3. **`FrameProcessor`** — містить набір класичних графічних фільтрів (Invert, Blur, Canny Edge, Glitch Effect).
4. **`FaceDetector` (Новий модуль рівня 2)** — інкапсулює логіку роботи нейромережі та керує фоновим потоком:
   * **Main Thread (UI):** Читає кадр, відправляє його копію у детектор, малює рамки за останніми відомими координатами та миттєво виводить зображення на екран (забезпечуючи стабільні 30-60 FPS).
   * **Worker Thread (Background):** `std::thread`, який безкінечно крутиться у фоні. Чекає на новий кадр, виконує `net.forward()`, і як тільки знаходить обличчя — оновлює спільну змінну з координатами.
   * **Синхронізація:** Для безпечного обміну даними між потоками (щоб один потік не почав читати координати, поки інший їх ще перезаписує) використовується `std::mutex`. Зупинка потоку при виході з програми контролюється через `std::atomic<bool>`.

## Вимоги до середовища (Environment)
* **Операційна система:** Ubuntu 24.04 LTS (або аналогічний Linux).
* **Компільтор:** GCC / G++ (стандарт C++17).
* **Система збірки:** CMake (3.10+), Make.
* **Бібліотеки:** OpenCV 4.x (модулі `core`, `videoio`, `highgui`, `imgproc`, `dnn`), стандартна бібліотека потоків `<thread>`, `<mutex>`.
